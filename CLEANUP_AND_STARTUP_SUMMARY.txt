DEPLOYMENT SCENARIOS - QUICK REFERENCE
================================================================================

LOCAL TESTING:
  ./start_server.sh --host 127.0.0.1 --port 8443
  - Only accessible from your machine
  - No privileges needed
  - Perfect for development

INTERNAL LAN:
  ./start_server.sh --host 192.168.1.100 --port 443
  - Agents on same network can connect
  - Requires root for port 443
  - Alternative: use --port 8443

INTERNET FACING:
  ./start_server.sh --host 0.0.0.0 --port 443
  - Listens on all interfaces
  - Requires root/admin
  - Configure agents with public IP

DOCKER/CONTAINER:
  ./start_server.sh --host 0.0.0.0 --port 5000
  - Port maps to host in docker-compose
  - No special privileges needed
  - Flexible port assignment

REVERSE PROXY:
  ./start_server.sh --host 127.0.0.1 --port 8443
  - Server runs locally
  - Nginx/Apache in front
  - Agents connect to proxy

MULTIPLE INSTANCES:
  ./start_server.sh --port 8001 &
  ./start_server.sh --port 8002 &
  - Each runs independently
  - Separate session management
